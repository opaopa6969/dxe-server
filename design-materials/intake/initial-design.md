# dxe-server 初期設計 — DGEセッション出力

- Date: 2026-04-04
- Source: DRE-toolkit の「DRE して」問題から派生
- Session: DxE Visualization Server 設計 DGE (design-review)

---

## 背景・動機

DxEシリーズ（DGE/DDE/DRE）が揃ってきたことで、**状態と決定を横断的に可視化する層**が必要になってきた。

- DRE はステートマシン（FRESH→INSTALLED→OUTDATED→CUSTOMIZED）を持つが、ターミナルではグラフィカル表示が難しい
- DGE は Gap をセッションファイル（.md）に蓄積するが、ファイルは人間が読む前提で構造的に集計できない
- DDE のドキュメント補完状態も同様

DxE-suite に `npx dxe ui` を追加する案もあったが、**複数プロジェクト横断集約**まで考えると専用パッケージが正解と判断。

---

## アーキテクチャ決定（ADR）

### ADR-001: 別パッケージとして分離する

**決定:** `@unlaxer/dxe-server` として独立パッケージ化。DxE-suite には統合しない。

**理由:**
- DxE-suite はインストール管理に集中すべき（カーネル役）
- dxe-server は可視化層（GUI層）。インストールは任意
- 将来の機能拡張（チーム共有サーバー化など）に対応しやすい

**起動:** `npx dxe-server start` → ローカルサーバー起動 → ブラウザが開く

---

## 可視化対象（2種類）

### 1. ステート可視化（State View）

現在の断面スナップショット。

| 対象 | 表示内容 |
|------|---------|
| DRE | FRESH / INSTALLED / OUTDATED / CUSTOMIZED の状態。どのファイルがカスタマイズ済みか |
| DGE | 各プロジェクトの Gap 一覧。Active / Void / Archived の分布 |
| DDE | ドキュメント補完状態（未補完ページ数など） |

### 2. 決定可視化（Decision View）

時系列の選択履歴。

| 対象 | 表示内容 |
|------|---------|
| DGE | どのGapをいつ・なぜ解決したか。Severity別トレンド |
| DRE | いつ install/update したか。何のファイルを変更したか |

---

## Gap 一覧（設計DGEセッションから）

| # | Gap | Severity |
|---|-----|----------|
| 1 | 「ステート」（現在断面）と「決定」（時系列履歴）は別概念——設計で分ける必要がある | High |
| 2 | APIサーバーの本質価値は「複数プロジェクト横断集約」——一人・一プロジェクトならファイルで十分 | Critical |
| 3 | 「誰がいつ何のために見るか」未定義——用途でサーバー要否すら変わる | Critical |
| 4 | 可視化層は `dxe-server` として別パッケージ分離。DxE-suiteはインストール管理に集中 | High |

## 未解決の重要問題（次のDGEセッションで深掘り推奨）

### Gap #3 深掘り: 誰がいつ使うか

以下を明確にする必要がある:

1. **ユーザープロファイル**
   - 一人開発者 — 自プロジェクトの状態確認
   - チームリード — 複数プロジェクトの横断確認
   - マネージャー — 週次レポート

2. **使用頻度**
   - 毎日見るダッシュボード → 常駐サーバーが必要
   - 週次確認 → `npx dxe-server start` のオンデマンドで十分
   - インストール時のみ → CLI出力で代替可能

3. **使用頻度が答えを決める理由**
   - 毎日: 常駐デーモン + 状態変化の通知
   - 週次: オンデマンド起動 + レポート生成
   - 単発: CLI の `dxe status` で十分（サーバー不要）

## DxEシリーズ全体像（更新版）

```
DGE-toolkit   → 設計Gap抽出（会話劇）
DDE-toolkit   → ドキュメント補完
DRE-toolkit   → rules/skills 配布・管理
─────────────────────────────────────
DxE-suite     → 3本のインストール管理 (npx dxe install)
dxe-server    → 状態・決定の可視化 API + ブラウザUI (optional)
               ├ DRE ステート可視化
               ├ DGE Gap ライフサイクル
               └ 複数プロジェクト横断集約
```
