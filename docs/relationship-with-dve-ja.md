# DVE との関係

DVE（Decision Visualization Engine）は DxE-suite の一部です（`@unlaxer/dve-toolkit`、`v4.2.0`）。このドキュメントでは DVE がすでに実装していること、DVE が止まるところ、dxe-server が価値を追加できる可能性があるところを整理します。

---

## DVE がすでに実装していること

DVE は DxE 決定グラフの**完全な可視化スタック**です。スタブではありません。

### バックエンド（`dve/kit/server/api.ts`、`localhost:4174`）

| エンドポイント | 機能 |
|---|---|
| `POST /api/scan` | DxE 対応プロジェクトを発見するためにディレクトリツリーをスキャン（DGE、DDE、DRE、DVE の存在を検出） |
| `POST /api/register` | 発見したプロジェクトをグラフに登録 |
| `POST /api/annotations` | アノテーションファイルをプロジェクトに書き戻し |
| `GET /api/graph` | 完全な決定グラフを JSON として返す |
| `GET /api/health` | ヘルスチェック |

### 状態検出（`dve/kit/parser/state-detector.ts`）

DVE は DRE インストール状態の完全な検出を実装しています：

- `FRESH` — `.claude/` が存在しない
- `INSTALLED` — DRE バージョンファイルが存在し、キットバージョンと一致
- `OUTDATED` — ローカルバージョンがキットバージョンより古い
- `CUSTOMIZED` — `.claude/` の任意のファイルがキットソースと異なる

これは元の dxe-server 設計が可視化を計画していたすべての DRE 状態をカバーしています。

### フロントエンド（`dve/app/src/`、Preact + Cytoscape）

- インタラクティブグラフ: Session → Gap → DD → Spec をクリック可能なノードで表示
- `ScanView.tsx` — `/api/scan` を呼び出すプロジェクト発見 UI
- ノードクラスタリング、フィルタリング、アノテーションオーバーレイ
- Vite が `localhost:4173`（開発）/ `localhost:4174`（プレビュー + API 同一ポート）で配信

---

## DVE がカバーしていないこと

| Gap | 詳細 |
|---|---|
| **DDE 内部補完状態** | DVE はプロジェクトごとに `hasDDE: boolean` を記録するだけです。どの DDE ページが完了か、どの用語が抽出済みか、どのグロサリーリンクが欠落しているかを可視化しません。 |
| **DDE 時系列進捗** | DDE 補完率の時系列履歴はありません |
| **週次ダイジェストレポート** | スケジュール実行またはオンデマンドのナラティブレポート形式はありません |
| **DDE 中心のビュー** | DVE は決定グラフ（DGE 中心）を軸に構成されています。DDE は第一級ビューではなく周辺の属性です。 |

これら4つのギャップが dxe-server にとって**唯一の非重複テリトリー**です。

---

## 率直な重複評価

dxe-server が元の完全スコープ（DRE 状態ビュー + DGE Gap ライフサイクル + マルチプロジェクトスキャン）を実装すると、**DVE のコア機能を重複実装**することになります。これは実行可能な道ではありません。

現実的なパスは：

1. **DVE に吸収** — DVE の Preact アプリに新しいビューとして DDE 可視化を追加する。新しいサーバー不要。（ADR-002 Option A）
2. **DDE 専念の dxe-server** — DDE 可視化のギャップのみを実装する。それ以外はすべて DVE に委任することを明示する。（ADR-002 Option B）
3. **併存** — dxe-server を異なる UX モデル（例：静的レポート生成、常駐サーバーなし）を持つ軽量コンパニオンとする。（ADR-002 Option C）

---

## 差別化候補

dxe-server が Option B または C を選択する場合、DVE が容易に吸収できない差別化要素：

| 候補 | DVE が容易に吸収できない理由 |
|---|---|
| DDE 補完率ダッシュボード | DVE のグラフモデルはノード/エッジ; 補完率は集計メトリクス |
| 週次 Markdown ダイジェストレポート | DVE はインタラクティブ UI; 静的レポートエクスポートはその UX モデルの外 |
| `dde status` ターミナル出力（ブラウザ不要） | DVE はブラウザが必要; ヘッドレスステータスは異なるユースケース |
| プロジェクト横断 DDE トレンド（時系列の補完進捗） | DVE グラフは現在状態のみ; 時系列は新しいデータモデルが必要 |

---

## 結論

DVE は DGE/DRE/決定グラフの可視化への答えです。**dxe-server の唯一の正当なスコープは DDE 可視化と DDE 中心のレポーティングです。**

dxe-server の実装は：
1. DVE を置き換えたり競合したりしないことを明示する
2. 決定グラフ機能のためにユーザーを DVE に誘導する
3. DDE 補完状態と週次レポート生成のみに集中する

保留中のスコープ決定については [ADR-002](decisions/ADR-002-scope-options.md) を参照してください。
